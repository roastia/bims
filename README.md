# 書店在庫管理システム(オフライン版)

サーバー不要・Pythonの複雑なインストール不要で使える、書店在庫管理システムです。

## 使い方(Mac)

1. このフォルダをまるごとデスクトップなど好きな場所に置く(またはこのリポジトリをclone/pull)
2. `start_mac.command` をダブルクリック
   - 初回に「開発元が未確認のため開けません」という警告が出た場合は、`start_mac.command` を**右クリック→「開く」→もう一度「開く」**を選んでください。次回からは普通にダブルクリックできます
   - 自動的にブラウザで開きます
3. 使い終わったら、開いた黒い画面(ターミナル)を閉じれば終了します

## Windowsで試す場合

`start_windows.bat` も同梱しています。実際のPython(本物の方。Microsoft Storeのダミーではなく https://www.python.org/ からインストールしたもの)が入っていれば同様に動作します。実行すると「Bookstore Server」という名前の最小化されたウィンドウが裏でサーバーを動かし続けます。使い終わったら、タスクバーからそのウィンドウを見つけて閉じてください。

## なぜ index.html を直接ダブルクリックしないのか

最近のブラウザ(Chrome・Safariとも)はfile://で直接開いたページに対して、データの保存機能(IndexedDB)を使えないようブロックする仕様になっています。そこで起動スクリプトは、Python標準機能だけを使ってこのフォルダをこのパソコン上だけで簡易に配信し、`http://localhost:8765` として開くようにしています。

- インターネット接続は不要(ISBNからの書誌自動取得のときだけ使います)
- 外部のサーバーには一切データを送りません。すべてこのパソコンの中だけで完結します
- pipやライブラリのインストールは一切行いません。Python本体だけあれば動きます

## データの保存場所

データはこのパソコンで開いた**ブラウザの中**(IndexedDB)に保存されます。同じパソコンの同じブラウザ・同じ方法(起動スクリプト経由)で開いたときだけ、前回のデータが引き継がれます。

## 他のパソコンでも同じデータを使いたい場合

画面上部メニューの「データ管理」から、エクスポート(.jsonダウンロード)→他のパソコンでインポート、という手順でデータを引き継げます。自動同期ではないため、閉店後などに定期的に行ってください。

## 機能

- 2段バーコード対応(上段ISBN→openBDで書誌自動取得、下段192/191コードから価格・分類を自動デコード)
- 入荷登録は「入荷条件(卸会社・仕入形態・返品期限・入荷日)」を最初に1回設定すれば、続けてスキャンするすべての本にその設定が引き継がれます
- **店舗間移動**: ある店舗の在庫を、卸会社・仕入形態・返品期限などの情報を保ったまま別の店舗へ移動できます
- フリータグによる棚・書架管理
- 卸会社×買切/委託ごとに冊数を分離管理し、売上時は「返品期限が近い委託在庫→古い入荷分」の順に自動引当
- 委託返品期限アラート・返品登録
- 入荷CSV・POS売上CSVの取込
- 簡易棚卸(差異は卸別内訳を崩さない「調整」として記録)
- ダッシュボード(在庫0一覧・返品期限アラート・売れ筋ランキング)
- 店舗切替(在庫/売上/タグは店舗ごと、書誌/卸会社マスタは共通)
- マスタ管理画面から、店舗・卸会社・書誌・タグの登録済みデータをすべて編集・削除可能
- データのエクスポート/インポート(バックアップ・他パソコンへの移行)
- 保存データに不整合があった場合、起動時に自動検出・修復します

## 開発について(GitHubで管理する場合)

このプロジェクトはビルド不要の静的ファイル(`index.html` 1つ + 起動スクリプト2つ)だけで構成されています。VS CodeなどでGit管理して編集し、コミット・プッシュするだけで運用できます。

### 初回セットアップ(このフォルダをGitリポジトリにする)

1. このフォルダをVS Codeで開く(File > Open Folder)
2. VS Code内のターミナル(Terminal > New Terminal)で以下を実行

```
git init
git add -A
git commit -m "Initial commit"
git branch -M main
```

3. GitHubで空の新規リポジトリを作成(READMEなどは追加しない)
4. 表示されるリポジトリURLを使って、続けて以下を実行

```
git remote add origin <リポジトリのURL>
git push -u origin main
```

以降はVS Codeの Source Control パネルから編集内容をコミット・プッシュできます。他のパソコンでは `git clone` または `git pull` するだけで最新版を取得できます。

(在庫データ自体はブラウザのIndexedDBに保存されているため、リポジトリには含まれません。データの引き継ぎは上記の「データ管理」画面からのエクスポート/インポートを使ってください。)
